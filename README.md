# Facial Recognition System

An end-to-end facial recognition system that combines deep learning feature extraction with machine learning classification to recognize human identities from unconstrained images. This project evolved from an initial prototype into a significantly more reliable system after identifying and resolving an Out-of-Distribution (OOD) evaluation issue, improving prediction performance from **2/5 (40%)** to **4/5 (80%)**.

---

# Why I Built This

Modern facial recognition systems are expected to work under varying lighting conditions, facial expressions, camera angles, and image quality. While training a model is relatively straightforward, building one that generalizes to unseen images is considerably more challenging.

The objective of this project was not simply to recognize faces, but to understand the engineering challenges involved in developing a reliable facial recognition pipeline similar to those used in authentication systems such as Face ID.

---

# Engineering Problem

The first version of this project achieved only **40% prediction accuracy (2 correct predictions out of 5 test images)**.

Instead of immediately changing models or tuning hyperparameters, I investigated **why** the system was failing.

The root cause was not the classifier.

It was the **evaluation data**.

The testing images were significantly different from the distribution of images the model had learned during training, causing an **Out-of-Distribution (OOD)** problem.

After redesigning the evaluation approach to better represent the training distribution, the same recognition pipeline achieved **80% prediction accuracy (4 correct predictions out of 5)**.

This project reinforced one of the most important engineering lessons in machine learning:

> Improving data quality and evaluation methodology can have a greater impact than replacing the model itself.

---

# Solution Overview

The system follows a hybrid Deep Learning + Machine Learning pipeline:

1. Detect faces using **MTCNN**
2. Crop and preprocess detected faces
3. Extract discriminative facial embeddings using **EfficientNet Lite0**
4. Train a **Support Vector Machine (SVM)** classifier on the extracted embeddings
5. Predict identities together with confidence scores

Separating feature extraction from classification allows the feature extractor and classifier to evolve independently while keeping the pipeline computationally efficient.

# Model Architecture

## Face Detection

- MTCNN
- Detects facial bounding boxes
- Crops the detected face

## Feature Extraction

- EfficientNet Lite0 (TensorFlow Hub)
- Converts each detected face into a numerical embedding representing facial characteristics.

## Classification

- Support Vector Machine (Linear Kernel)
- Hyperparameter tuning performed to improve classification performance.

# Dataset

**Dataset:** Labeled Faces in the Wild (LFW – Deepfunneled)

The dataset contains real-world facial images collected under unconstrained conditions, making it significantly more challenging than controlled face datasets.

Images include variations in:

- Pose
- Lighting
- Facial expression
- Background
- Image quality

These variations make LFW a suitable benchmark for evaluating practical facial recognition systems.

---

# Key Engineering Decisions

### Why MTCNN?

MTCNN provides reliable face localization while handling facial alignment effectively before feature extraction.

---

### Why EfficientNet?

Rather than training a CNN from scratch, EfficientNet provides powerful transferable visual representations while significantly reducing training time.

---

### Why SVM instead of a Neural Network classifier?

Instead of adding another neural network classifier, facial embeddings generated by EfficientNet were classified using an SVM.

For relatively smaller datasets like LFW, SVM often generalizes better and reduces overfitting while remaining computationally efficient.

## Project Evolution

The initial version of the system achieved only **40% accuracy (2/5 correct predictions)** during demonstration.

Rather than immediately replacing the model, I investigated the failure cases and discovered that the evaluation images suffered from an **Out-of-Distribution (OOD)** issue. After redesigning the evaluation process, the same recognition pipeline achieved **80% accuracy (4/5 correct predictions)**.

This improvement demonstrates the importance of **data quality and evaluation methodology** in building reliable AI systems.
